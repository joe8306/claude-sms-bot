CLAUDE SMS BOT - SETUP STEPS (FLY.IO DEPLOY)
=============================================

WHAT THIS DOES
--------------
You text your Twilio number from your flip phone. Twilio HTTP-POSTs the
message to a Flask server running 24/7 on Fly.io. The server sends it to
Claude, gets a reply, and Twilio sends that reply back to you as an SMS.

Your PC does NOT need to be on. The Fly machine sleeps when idle and wakes
up in 1-3 seconds when a text comes in (within Twilio's 15s webhook window).

History persists on a Fly volume so context survives sleeps and restarts.


ONE-TIME SETUP
==============

1. INSTALL FLYCTL  (Windows PowerShell, run as your normal user)
   iwr https://fly.io/install.ps1 -useb | iex
   Then close + reopen your terminal so PATH picks up flyctl.

2. CREATE A FLY ACCOUNT
   flyctl auth signup
   (or `flyctl auth login` if you already have one)
   You'll need to add a credit card - Fly removed their truly-free tier,
   but this app costs ~$2/mo or less because the machine auto-stops.

3. GET YOUR KEYS READY (you'll paste these in step 5)
   - Anthropic API key:  https://console.anthropic.com/settings/keys
     (separate from Claude Code billing - $5 minimum top-up)
   - Twilio Auth Token:  https://console.twilio.com (front page)

4. LAUNCH THE APP  (from this folder)
   cd C:\Users\chaim\Downloads\CCode\claude_sms_bot
   flyctl launch --no-deploy --copy-config
       - "Would you like to copy its configuration to the new app?" -> Yes
       - It may rename the app if "claude-sms-bot" is taken; accept the suggestion
       - Decline Postgres, Redis, Sentry - don't need them
       - Pick region "iad" (Ashburn) or whichever it suggests

5. SET YOUR SECRETS
   flyctl secrets set ANTHROPIC_API_KEY=sk-ant-... TWILIO_AUTH_TOKEN=...
   (paste the real values - they're encrypted at rest, never visible again)

6. CREATE THE PERSISTENT VOLUME
   flyctl volumes create claude_sms_data --size 1 --region iad --yes
   (1 GB is way more than enough; costs ~$0.15/mo)

7. DEPLOY
   flyctl deploy
   First build takes 2-3 minutes. When it finishes, run:
   flyctl status
   Copy the Hostname (looks like: claude-sms-bot-xxxx.fly.dev)

8. POINT TWILIO AT IT  (one time, URL never changes)
   console.twilio.com -> Phone Numbers -> your number -> Messaging Configuration
   "A message comes in" webhook:  https://YOUR-APP.fly.dev/sms
   Method: HTTP POST
   Save.

9. TEXT YOUR TWILIO NUMBER FROM YOUR FLIP PHONE
   First message wakes the machine (~2s delay), follow-ups are instant.

NOTE on Twilio verification: as long as you only text yourself (your flip
phone is in Twilio's "Verified Caller IDs" list), you don't need A2P 10DLC
registration. Add your flip number at:
console.twilio.com/us1/develop/phone-numbers/manage/verified


DAY-TO-DAY
==========

View live logs:           flyctl logs
Update the bot code:      edit bot.py, then `flyctl deploy`
Change a secret:          flyctl secrets set KEY=newvalue   (auto-redeploys)
Wipe all conversations:   flyctl ssh console -C "rm /data/conversations.json"
Wipe one number's chat:   curl -X POST https://YOUR-APP.fly.dev/reset -d "From=+15551234567"
                          (no signature check on /reset - lock this down later if it matters)


COSTS
=====

Fly.io:
   Machine (256MB shared CPU, auto-stop):   ~$0-2/mo  (only billed when awake)
   Volume (1 GB):                           ~$0.15/mo
   Bandwidth:                               negligible
   Total:                                   under $3/mo

Twilio (your $20 credit covers months of use):
   Phone number:     $1.15/month
   Receive SMS:      $0.0079 each
   Send SMS:         $0.0083 each
   1000 round-trips: ~$16

Anthropic (Sonnet 4.6: $3 per 1M input / $15 per 1M output):
   Typical SMS exchange: ~500 in + 200 out = $0.0045
   1000 round-trips: ~$4.50

GRAND TOTAL for 1000 messages/month: ~$25
   ($3 Fly + $1.15 Twilio number + $8 Twilio SMS + $4.50 Claude + headroom)
   For your 500 msgs target: ~$15/mo all-in.


TROUBLESHOOTING
===============

"Application Error" when texting:
   flyctl logs           # see the actual error
   Common: forgot to set ANTHROPIC_API_KEY or TWILIO_AUTH_TOKEN as secrets

Webhook returns 403:
   Twilio auth token in the secret doesn't match the one Twilio is using
   to sign requests. Re-copy from console.twilio.com and `flyctl secrets set`.

Reply takes 5+ seconds on first text after a long pause:
   Normal - the machine was stopped. It wakes on the incoming HTTP request.
   Subsequent texts are instant until ~5 minutes of idle.

Want it to never sleep:
   In fly.toml change `min_machines_running = 0` to `1`. Costs ~$2/mo more.
